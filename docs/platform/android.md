# Android 速查

- **架构**：MVVM — `ui/` → `domain/repository/` → `data/network/`（语音：`data/speech/`、`data/audio/`）
- **依赖注入**：手写 `AppContainer`（后续可换 Hilt/KSP 等）
- **构建变体**：`dev`（HTTP、本地 API）/ `prod`（HTTPS、生产 API）；`BuildConfig.API_BASE_URL`、`HEALTH_URL`
- **配置**：`local.properties` 中 `sdk.dir`；可选 `local.server.host`（Gradle 默认如 `192.168.1.1`）、`local.server.port`（默认 `8000`）
- **首页 / 语音**：`ui/home/` — 按住说话与对话转写；录音时**上划约 150dp** 进入取消区（红色渐变 +「Release to cancel」），松手则 **`cancelSession()`** 丢弃本次转写且不触发 Agent；滑回下方再松手为正常 **`stopSession()`** 发送。`domain/speech/SpeechTranscriber`（含 `cancelSession`）+ `data/speech/`（OkHttp WebSocket）；`AudioRecorder`（16 kHz PCM）。`buildSpeechWebSocketUrl(host)` 只生成无凭据的 `ws://<host>:<port>/api/v1/speech/ws`；`RemoteSpeechTranscriber` 经 `SpeechTokenProvider` 在升级请求上加 **`Authorization: Bearer`**。握手 **401/403** 时调 `/auth/refresh` 后 **重连一次**；仍失败则 `clearAuth()`，`ui/main/MainRoute.kt` 根据 `isLoggedIn` 回登录。服务端 `/api/v1/speech/ws` 支持 Bearer，并兼容 `?access_token=`
- **图片（待发送）**：`PickMultipleVisualMedia`（最多 9 张）选图后**不立即上传**；`SpeechViewModel.pendingImageUris` 持有本地 `Uri`，`PendingImagesBar`（Coil 缩略图）展示 **发送** 或 **单张移除**。点发送：逐张 **`POST /api/v1/media`** 后插入用户消息与占位助手回复。按住说话结束且仍有待发送图：先上传再 **`/api/v1/agent/chat`**，请求体带 **`media_ids`**（与 `AgentSseClient.streamChat` 对齐）。
- **Agent 回复 UI**：`/api/v1/agent/chat` SSE 在 `SpeechViewModel` 中聚合为助手气泡；`AssistantChatComponents` 对缓冲区做逐字显示（`TypewriterAssistantText`），气泡下方为「处理中」→「处理完成」的 `AnimatedContent` 过渡（仅 `ChatMessage.showAgentStatusRow` 的语音 Agent 占位消息）。请求带 **`require_confirmation=true`**（`AgentSseClient`）；若流末尾收到 **`proposed_actions`**，则保持该条助手消息为 pending，弹出 **`ConfirmActionsSheet`**（多选、`ModalBottomSheet`），用户确认后 **`POST /api/v1/agent/execute-actions`**，取消或划掉则仅结束 pending 状态。SSE 首包 **`event: thread`** 时把 `thread_id` 写入 DataStore（`UserPreferencesRepository.agentThreadId`），后续请求携带 **`thread_id`**；`SpeechViewModel` 在 **`loadHistoryFromServer()`** 中 **`GET /api/v1/agent/threads/{id}/history`** 恢复列表（登出时 `clearAuth` 会清除 `agent_thread_id`）。
- **多租户 API**：注册/登录返回含 `tenant_id` 的 JWT；租户相关 HTTP 请求带 **`Authorization: Bearer`**（见 `di/AppContainer.kt` 中 `AuthInterceptor`）
- **日程（Schedule）**：列表项时间范围来自 Todo 的 **`scheduled_at`**（若服务端有值，表示用户意图日程时刻）；否则回退到 **`created_at`**（创建时间），结束时间为开始时间 + 30 分钟（见 `ui/schedule/TodoScheduleMapping.kt`）。API 时刻为带时区的 ISO  instant，展示前会 **`atZoneSameInstant(ZoneId.systemDefault())`** 转为设备本地墙钟时间，避免把 UTC 误当本地时间。
