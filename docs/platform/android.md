# Android 速查

- **架构**：MVVM — `ui/` → `domain/repository/` → `data/network/`（语音：`data/speech/`、`data/audio/`）；主 tab 下 Home 对话列表的 **`LazyListState` 挂在 `MainScreen`**（`remember`），切 tab 再回 Home 时不会重建列表滚动状态，避免从顶动画滚到底的卡顿；自动贴底逻辑与 **`scrollToItem`** 也在 `MainScreen` 的 **`LaunchedEffect`** 中，仅在当前为 Home tab 时执行。
- **依赖注入**：手写 `AppContainer`（后续可换 Hilt/KSP 等）
- **构建变体**：`dev`（HTTP、本地 API）/ `prod`（HTTPS、生产 API）；`BuildConfig.API_BASE_URL`、`HEALTH_URL`
- **配置**：`local.properties` 中 `sdk.dir`；可选 `local.server.host`（Gradle 默认如 `192.168.1.1`）、`local.server.port`（默认 `8000`）
- **首页 / 语音**：`ui/home/` — 按住说话与对话转写；录音时**上划约 150dp** 进入取消区（红色渐变 +「Release to cancel」），松手则 **`cancelSession()`** 丢弃本次转写且不触发 Agent；滑回下方再松手为正常 **`stopSession()`** 发送。每次开始按住前 **`SpeechViewModel`** 会先 **`cancelSession()`** 清掉未结束的会话（否则 `RemoteSpeechTranscriber.startSession` 在非 `Idle` 时直接返回，可能表现为「发完一次语音后无法再录」）；离开 Home Tab 时 **`onHomeLeave()`** 同样 **`cancelSession()`**，避免半次手势把转写器卡住。一轮结束进入 **`Idle`** 后 **`SpeechViewModel`** 会调用 **`clearTranscriptIfIdle()`**，避免旧转写文本留在 **`StateFlow`** 中干扰下一轮。底部 **`+` 与麦克风** 叠放时 **`VoiceMicButton` 在上层**（`zIndex`），避免 **`ImageAddButton`** 抢走麦克风的触摸；**`VoiceMicButton`** 的 **`pointerInput` 挂在整块竖向区域**（含计时/提示文字占位），避免点到透明文字区时事件穿透到下方 **`LazyColumn`**；有待发图时 **`LazyColumn`** 底部 **`contentPadding`** 额外加高约 **`PendingImagesBar`**，选图返回后 **`clearFocus()`** 减轻系统返回后焦点/触摸链异常。`domain/speech/SpeechTranscriber`（含 `cancelSession`、`clearTranscriptIfIdle`）+ `data/speech/`（OkHttp WebSocket）；`AudioRecorder`（16 kHz PCM）。`buildSpeechWebSocketUrl(host)` 只生成无凭据的 `ws://<host>:<port>/api/v1/speech/ws`；`RemoteSpeechTranscriber` 经 `SpeechTokenProvider` 在升级请求上加 **`Authorization: Bearer`**。握手 **401/403** 时调 `/auth/refresh` 后 **重连一次**；仍失败则 `clearAuth()`，`ui/main/MainRoute.kt` 根据 `isLoggedIn` 回登录。服务端 `/api/v1/speech/ws` 支持 Bearer，并兼容 `?access_token=`
- **图片（待发送）**：点加号打开 **`AttachmentImageSheet`**（`ModalBottomSheet`，自底向上动画）：**相册** 走 `PickMultipleVisualMedia`（最多 9 张）、**拍照** 走 `TakePicture` + **`FileProvider`**（`res/xml/file_paths.xml`，`cache-path` 下 `camera/`）；**近期** 网格由 `RecentImages` 查 `MediaStore`（需 **`READ_MEDIA_IMAGES`**（API 33+）或 **`READ_EXTERNAL_STORAGE`**（≤32）），点缩略图加入待发送列表。加号在面板打开时显示为关闭。选图后**不立即上传**；`SpeechViewModel.pendingImageUris` 持有本地 `Uri`（去重、最多 **9** 张），**`PendingImagesBar`** 为浅色圆角矩形横条，叠在 **麦克风 / 加号一行下方**（不压在对话气泡上）：**缩略图 + 右上角移除**、**「+」继续选图**（再开底部面板）、**发送**。点发送：逐张 **`POST /api/v1/media`**，再 **`POST /api/v1/agent/chat`**（默认文案 `image_send_default_message` + **`media_ids`**），用户气泡用 **`ChatMessage.imageUris`** 展示本地缩略图。按住说话结束且仍有待发送图：先上传再 **`/api/v1/agent/chat`**（转写文本 + **`media_ids`**），用户气泡同时带 **`imageUris`**。历史恢复：**`GET /api/v1/agent/threads/{id}/history`** 返回的每条可含 **`media`**（`id`、`content_type`），`SpeechViewModel` 转为 **`mediaUrls`**（`…/api/v1/media/{id}`）；**`ChatBubble`** 用 **`AppContainer.imageLoader`**（与 API 同鉴权的 OkHttp）加载远程图。**`CAMERA`** 用于拍照。
- **文案与 i18n**：用户可见文案集中在 `app/src/main/res/values/strings.xml`（默认英文），简体中文在 `values-zh-rCN/strings.xml`。设置页可选择 **跟随系统 / English / 简体中文**；选择经 DataStore（`UserPreferencesRepository` 键 `app_locale`）持久化，启动时在后台协程 `hydrateFromDataStore()` 与写入后调用 **`AppCompatDelegate.setApplicationLocales`**（`i18n/AppLocale.kt` 映射到 `LocaleListCompat`；`SYSTEM` 用空列表清除应用内覆盖）。**主界面须为 `AppCompatActivity` + `Theme.AppCompat.*`（见 `MainActivity`、`themes.xml`）**，否则仅 `ComponentActivity` + 纯系统 Material 主题时，应用内语言切换可能不生效。`AndroidManifest` 的 `application android:localeConfig` 指向 `res/xml/locales_config.xml`（`en`、`zh-CN`），与系统「应用语言」列表对齐（Android 13+）。**扩展新语言**：增加 `values-xx/`、`locales_config.xml` 中声明 locale、在 `AppLocale` 增加枚举分支与 `toLocaleListCompat()`。ViewModel 层用 `Application.getString()` 或 Compose `stringResource`，避免硬编码。
- **Agent 回复 UI**：`/api/v1/agent/chat` SSE 在 `SpeechViewModel` 中聚合为助手气泡；`AssistantChatComponents` 对缓冲区做逐字显示（`TypewriterAssistantText`），气泡下方为「Processing…」→「Done」的 `AnimatedContent` 过渡（仅当前**最新一条**且 `ChatMessage.showAgentStatusRow` 的语音 Agent 占位消息；新对话开始后「Done」不再显示）。请求带 **`require_confirmation=true`**（`AgentSseClient`）；若流末尾收到 **`proposed_actions`**，则保持该条助手消息为 pending，弹出 **`ConfirmActionsSheet`**（多选、`ModalBottomSheet`），用户确认后 **`POST /api/v1/agent/execute-actions`**。用户在清单上点取消、未选任何项点确认、或划掉 sheet：该条助手气泡改为取消文案（`agent_chat_action_cancelled`），`ChatMessage.agentCancelled = true`，气泡下显示**持久**的「已取消 / Cancel」状态行（`AssistantReplyCancelledStatusRow`，新对话后仍保留）。流中可出现 **`event: clarification`**（`ask_user_questions`），`AgentSseClient` 解析为 **`AgentSseEvent.Clarification`**；追问正文通常已含在 token 流中，事件供结构化扩展。SSE 首包 **`event: thread`** 时把 `thread_id` 写入 DataStore（`UserPreferencesRepository.agentThreadId`），后续请求携带 **`thread_id`**；`SpeechViewModel` 在 **`loadHistoryFromServer()`** 中 **`GET /api/v1/agent/threads/{id}/history`** 恢复列表（登出时 `clearAuth` 会清除 `agent_thread_id`）；历史接口不还原 `agentCancelled`，仅本会话内存态。
- **多租户 API**：注册/登录返回含 `tenant_id` 的 JWT；租户相关 HTTP 请求带 **`Authorization: Bearer`**（见 `di/AppContainer.kt` 中 `AuthInterceptor`）。**`AuthInterceptor`** 仅当路径以 **`/api/v1/auth/`** 为前缀时不加 Bearer（登录/注册/刷新等）；避免用子串匹配误伤其它路径。
- **日程（Schedule）**：列表项时间范围来自 Todo 的 **`scheduled_at`**（若服务端有值，表示用户意图日程时刻）；否则回退到 **`created_at`**（创建时间）。事件时长取 **`estimated_minutes`**（AI 创建 Todo 时自动估算，如"吃饭"→30min、"开会"→60min）；若该字段为空则回退到默认 30 分钟（见 `ui/schedule/TodoScheduleMapping.kt`）。API 时刻为带时区的 ISO  instant，展示前会 **`atZoneSameInstant(ZoneId.systemDefault())`** 转为设备本地墙钟时间，避免把 UTC 误当本地时间。`ScheduleScreen` 横向分页：页 0 为月历（`ScheduleMonthCalendar`），**仅有日程的日期可点击**，点击后 **`animateScrollToPage(1)`** 回到时间序列表（`ScheduleEventList`），并 **`scrollToItem`** 滚到对应日期分组。
- **账单（Bills）**：`GET /api/v1/bills` 经 **`BillRepositoryImpl`** → **`BillsViewModel`** → **`BillsScreen`**；按 **`billed_at`**（无则 **`created_at`**）分组日期列表；列表金额用 **`NumberFormat`** + **`Currency.getInstance("CNY")`**（与默认 **`Locale`**）格式化；收入/支出强调色见 **`ui/theme/SemanticColors.kt`**（`FinanceColors`）。Agent 确认清单中 **`ProposedAction`** 含 **`target`**（`todo` \| `bill`）、**`display_amount`**（账单摘要）、**`display_scheduled_at`**（账单 `billed_at` 墙钟摘要），**`ConfirmActionsSheet`** 在标题下同时展示金额行与时间行（若有）；确认按钮与操作类型色块使用 **`ActionColors`**。
